ARGen Help


The Review & Generate view shows any conflicts with the project configuration and is where developers will find the Generate Project button.

The Quick Resolve button will deselect class and user interface generation for tables and views with conflicts. This isn't always necessary to resolve issues, but is a fast and easy way to do so.

Common Conflicts:
These common messages can sometimes be resolved as described.

"Table has User Interface selected but does not generate ActiveRecord classes."
This message means the checkbox selection for the table is disabled in Class Setup, but selected in User Interface. The user interface components require the ActiveRecord classes. This can be resolved by either disabling User Interface generation for this table, or enabling class generation.

"Table / field cannot have a space in the name."
"Table / field has a illegal character in then name"
Table and column names are directly converted to class names, so names must meet the Xojo language naming rules.
https://docs.xojo.com/UserGuide:Variables_and_Constants#Naming_Rules

"Table does not have a Primary Key."
The ActiveRecord core uses a table's primary key to track and update a record. Since these tables are incompatible with ActiveRecord, deselect the incompatible tables in the Class Setup view

"View Primary Key is not an Integer. Web 2.0 requires an Integer Primary Key."
The Web 2.0 user interface code makes use of WebDataSource, which currently assumes the primary key is an integer. String GUID primary keys and Views without a primary key are incompatible with Xojo Web 2.0. These issues can be resolved by deselecting user interface generation for offending tables and views. Note that Quick Resolve will deselect the class generation for offending tables / views, which isn't necessary to resolve these. A feedback ticket regarding WebDataSource integer primary key requirements can be found here
